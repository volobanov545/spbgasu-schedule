# GASUCHKA — расписание СПбГАСУ 3-СУЗСс-2

Автоматическая система мониторинга расписания, посещаемости и аттестаций для группы 3-СУЗСс-2.  
Данные берутся из личного кабинета [portal.spbgasu.ru](https://portal.spbgasu.ru).

## Что умеет

- Парсит расписание (прошлая неделя + текущая + 2 следующие) → синхронизирует в Яндекс.Календарь
- Отслеживает изменения расписания (переносы, замены, отмены) → уведомление в Telegram-канал [@gasu4ka](https://t.me/gasu4ka)
- Парсит журналы посещаемости и аттестаций → личное уведомление при новом пропуске или изменении аттестации
- Запускается автоматически 3 раза в день через GitVerse CI/CD
- **Telegram-бот [@gasu4ka_bot](https://t.me/gasu4ka_bot)** — личная статистика для каждого студента группы

## Расписание обновлений

| Время МСК | Cron |
|-----------|------|
| 07:00 | `0 4 * * *` |
| 13:30 | `30 10 * * *` |
| 21:00 | `0 18 * * *` |

## Telegram-бот — как подключиться

1. Напиши [@gasu4ka_bot](https://t.me/gasu4ka_bot) команду `/start`
2. Введи логин от портала (студенческий номер, например `24001234`)
3. Введи пароль от портала
4. Опционально: подключи Яндекс.Календарь (инструкция ниже)
5. Дождись подтверждения от администратора

После подтверждения в чате появится постоянная клавиатура с кнопками:
- **📋 Аттестации** — посещаемость и аттестации в реальном времени (парсит портал ~20 сек)
- **📅 Яндекс.Календарь** — подключить Яндекс.Календарь (кнопка исчезает после подключения)
- **❌ Удалить аккаунт** — удаляет аккаунт и календарь «СПбГАСУ» из Яндекса

### Как получить пароль приложения Яндекса

1. Открой [id.yandex.ru](https://id.yandex.ru)
2. Безопасность → Пароли приложений
3. Нажми «Создать пароль» → выбери «Другое»
4. Скопируй пароль из 16 символов — он вводится в боте

## Уведомления

| Событие | Куда приходит |
|---------|--------------|
| Изменилось расписание (замена, перенос, отмена) | Канал [@gasu4ka](https://t.me/gasu4ka) |
| Новый пропуск в журнале | Личка владельцу |
| Изменилась аттестация | Личка владельцу |
| Вырос % отсутствий | Личка владельцу |

Уведомления отправляются только при реальных изменениях — не при каждом запуске CI.

## Архитектура

```
GitVerse CI (07:00 / 13:30 / 21:00 МСК)
  ├─ parse_portal.py   → schedule.ics
  ├─ parse_journals.py → journals_state.json
  ├─ notify.py         → pending_notification.json
  ├─ sync_yandex.py    → Яндекс.Календарь (владелец)
  └─ git push → GitVerse + GitHub (зеркало)

GitHub Actions (при изменении pending_notification.json)
  └─ send_notifications.yml → Telegram
       ├─ изменения расписания → канал @gasu4ka
       └─ пропуски / аттестации → личка владельцу

Amvera (бот, работает 24/7)
  └─ bot.py — регистрация, кнопочная клавиатура, управление пользователями
       ├─ parse_journals.py → Playwright → портал (личные данные студента)
       └─ sync_yandex.py   → CalDAV → Яндекс (личный календарь студента)
```

**Почему GitVerse CI + GitHub Actions, а не одно место:**  
GitVerse runners находятся в РФ — Telegram там заблокирован. GitHub runners работают за рубежом — Telegram доступен.

## Файлы проекта

| Файл | Назначение |
|------|------------|
| `parse_portal.py` | Парсер расписания (Playwright → portal.spbgasu.ru) |
| `parse_journals.py` | Парсер журналов и аттестаций; `parse_lk_quick(login, pass)` для бота |
| `sync_yandex.py` | Синхронизация ICS → Яндекс CalDAV; `delete_yandex_calendar()` при удалении аккаунта |
| `notify.py` | Уведомления: расписание → канал, журналы → личка |
| `bot.py` | Telegram-бот: регистрация, кнопки, управление пользователями (admin) |
| `db.py` | SQLite + Fernet-шифрование паролей пользователей |
| `Dockerfile` | Образ для Amvera (python:3.12-slim + Playwright Chromium) |
| `schedule.ics` | Текущее расписание |
| `journals_state.json` | Текущее состояние журналов |
| `.gitverse/workflows/update_schedule.yml` | GitVerse CI pipeline |
| `.github/workflows/send_notifications.yml` | GitHub Actions → Telegram |

## Команды бота (для администратора)

| Команда | Описание |
|---------|----------|
| `/users` | Список всех пользователей с кнопками одобрить/забанить/удалить |
| `/approve <id>` | Одобрить заявку |
| `/deny <id>` | Отклонить и удалить |
| `/ban <id>` / `/unban <id>` | Заблокировать / разблокировать |
| `/remove <id>` | Удалить пользователя |
| `/sendhtml` | Прислать debug_lk.html (последний парсинг портала) |
| `/sendpng` | Прислать скриншот страницы /lk/ |
