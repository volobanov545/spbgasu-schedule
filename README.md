# GASUCHKA — расписание СПбГАСУ 3-СУЗСс-2

Автоматическая система мониторинга расписания, посещаемости и аттестаций для группы 3-СУЗСс-2.  
Данные берутся из личного кабинета [portal.spbgasu.ru](https://portal.spbgasu.ru).

## Что умеет

- Парсит расписание (прошлая неделя + текущая + 2 следующие) → синхронизирует в Яндекс.Календарь
- Отслеживает изменения расписания (переносы, замены, отмены) → уведомление в Telegram-канал [@gasu4ka](https://t.me/gasu4ka)
- Парсит журналы посещаемости и аттестаций → личное уведомление при новом пропуске или изменении аттестации
- Запускается автоматически 3 раза в день через GitVerse CI/CD
- **Telegram-бот [@gasu4ka_bot](https://t.me/gasu4ka_bot)** — личная статистика для каждого студента группы

## Telegram-бот — как подключиться

1. Напиши [@gasu4ka_bot](https://t.me/gasu4ka_bot) или нажми кнопку **НАЧАТЬ**
2. Нажми кнопку **«📝 Зарегистрироваться»**
3. Введи логин от портала (студенческий номер, например `24001234`)
4. Введи пароль от портала
5. Опционально: подключи Яндекс.Календарь (инструкция ниже)
6. Дождись подтверждения от администратора

После подтверждения в чате появится постоянная клавиатура:
- **📋 Аттестации** — посещаемость и аттестации в реальном времени (~20 сек)
- **👥 Пользователи** — список пользователей с управлением *(только администратор)*
- **⚙️ Настройки** — подключить/отключить Яндекс.Календарь, удалить аккаунт

> **Пустой чат / очищена история:** нажми кнопку **≡** рядом с полем ввода — там есть `/start`.

### Как получить пароль приложения Яндекса

1. Открой [id.yandex.ru](https://id.yandex.ru)
2. Безопасность → Пароли приложений
3. Нажми «Создать пароль» → выбери «Другое»
4. Скопируй пароль из 16 символов — он вводится в боте

> Обычный пароль от Яндекса не подойдёт — нужен именно пароль приложения.

## Архитектура

```
GitVerse CI (07:00 / 13:30 / 21:00 МСК)
  ├─ parse_portal.py   → schedule.ics
  ├─ parse_journals.py → journals_state.json
  ├─ notify.py         → pending_notification.json
  ├─ sync_yandex.py    → Яндекс.Календарь (владелец)
  └─ git push → GitVerse + GitHub (зеркало)

GitHub Actions (при изменении pending_notification.json)
  └─ send_notifications.yml → Telegram
       ├─ изменения расписания → канал @gasu4ka
       └─ пропуски / аттестации → личка владельцу

Amvera (бот, работает 24/7)
  └─ bot.py — регистрация, клавиатура, управление пользователями
       ├─ parse_journals.py → Playwright → портал (личные данные студента)
       └─ sync_yandex.py   → CalDAV → Яндекс (личный календарь студента)
```

**Почему GitVerse CI + GitHub Actions, а не одно место:**  
GitVerse runners находятся в РФ — Telegram там заблокирован.  
GitHub runners работают за рубежом — Telegram доступен.

## Устройство бота (bot.py)

### Два вида клавиатур

| Тип | Где видна | Используется для |
|-----|-----------|-----------------|
| `ReplyKeyboardMarkup` | Прилипает к полю ввода навсегда | Главная навигация (Аттестации / Настройки) |
| `InlineKeyboardMarkup` | Прикреплена к конкретному сообщению | Действия: одобрить/забанить/удалить, настройки |

### Диалоги (ConversationHandler)

**reg_handler** — регистрация нового пользователя:
```
/start или кнопка «Зарегистрироваться»
  → WAIT_LOGIN → WAIT_PASSWORD → WAIT_YC_CHOICE
  → (да) → WAIT_YC_LOGIN → WAIT_YC_PASS → END
  → (нет) → END
```

**yc_handler** — подключение Яндекс.Календаря после регистрации:
```
⚙️ Настройки → «Подключить ЯК» или /connect_yandex
  → WAIT_YC2_LOGIN → WAIT_YC2_PASS → END
```

Порядок `app.add_handler()` важен: ConversationHandler-ы добавляются первыми и перехватывают апдейты раньше. Глобальный `CallbackQueryHandler(on_callback)` добавляется после — ловит только то, что не забрал ни один диалог.

### Особенности портала СПбГАСУ

Портал использует **Bitrix CMS + Vue SPA**:
- Авторизация через Bitrix на `/auth/`
- Дашборд — Vue-приложение на `/lk/`
- После Bitrix-логина нужно явно перейти на `/lk/` — тогда Vue подхватывает сессию
- Аттестации рендерит Vue — нужно ждать появления строк в таблице (`wait_for_function`)
- Сессия **не кешируется** — всегда делаем свежий логин; протухшие куки приводят к тому, что Vue показывает форму логина вместо данных

### Callback data

| callback_data | Кто вызывает | Что делает |
|--------------|-------------|-----------|
| `register` | кнопка «Зарегистрироваться» | Запускает диалог регистрации |
| `stats` | старые inline-меню | Вызывает cmd_stats |
| `unregister` | ⚙️ Настройки | Показывает подтверждение удаления |
| `confirm_unregister` | подтверждение | Удаляет аккаунт + календарь ЯК |
| `disconnect_yc` | ⚙️ Настройки | Удаляет ЯК без удаления аккаунта |
| `connect_yandex` | ⚙️ Настройки | Запускает yc_handler |
| `cancel_action` | подтверждения | Отмена |
| `approve:TID` | карточка заявки | Одобряет пользователя |
| `deny:TID` | карточка заявки | Отклоняет и удаляет |
| `ban:TID` | /users | Блокирует |
| `unban:TID` | /users | Разблокирует |
| `owner_remove:TID` | /users | Удаляет пользователя + ЯК |
